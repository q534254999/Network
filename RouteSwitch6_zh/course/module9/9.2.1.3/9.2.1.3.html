验证静态 NAT

用于验证 NAT 操作的一个有用命令是 show ip nat translations。此命令可用于显示活动的 NAT 转换。与动态转换不同，静态转换始终在 NAT 表中进行。图 1 使用上述配置示例显示此命令的输出。由于该示例是静态 NAT 配置，因此不论正在进行的是何种通信，转换都会在 NAT 表中进行。如果此命令是正在进行会话时发出的，则输出还会表示出外部设备的地址，如图 1 所示。

另一个有用命令是 show ip nat statistics。如图 2 所示，show ip nat statistics 命令显示有关总活动转换数、NAT 配置参数、地址池中地址数量和已分配地址数量的信息。

为了验证 NAT 转换是否正常工作，最好在测试前使用 clear ip nat statistics 命令清除任何之前转换的统计信息。

在与 Web 服务器通信之前，show ip nat statistics 命令显示无当前点击。在客户端与 Web 服务器建立会话后，show ip nat statistics 显示在内部（串行 0/0/0）接口上增加到五次点击。这将验证 R2 上是否发生了静态 NAT 转换。

图 1 显示没有任何活动会话时“show ip nat translations”命令的输出。外部地址为空。此外，还显示了有一个活动会话时“show ip nat translations”命令的输出。存在外部地址。这表明静态 NAT 始终存在 NAT 转换。图 2 显示了“clear ip nat statistics”命令以及紧跟的“show ip nat statistics”命令的输出，显示无点击。当 PC 与 Web 服务器建立会话时，“show ip nat statistics”命令的输出中显示了五个点击。
