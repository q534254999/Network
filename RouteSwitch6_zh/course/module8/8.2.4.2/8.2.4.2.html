验证路由器 DHCPv6 配置

无状态和有状态 DHCPv6 服务的路由器配置有很多相似之处，但也包括明显的差异。图 1 显示了这两类 DHCPv6 服务的配置命令。

有状态 DHCPv6

配置在有状态 DHCPv6 服务上的路由器使用 address prefix 命令提供编址信息。

对于有状态 DHCPv6 服务，使用 ipv6 nd managed-config-flag 接口配置模式命令。在本示例中，客户端忽略了 RA 消息中的编址信息，并与 DHCPv6 服务器通信以获取编址信息和其他信息。

无状态 DHCPv6

对于无状态 DHCPv6 服务，使用 ipv6 nd other-config-flag 接口配置模式命令。这会通知设备使用 SLAAC 获取编址信息，使用无状态 DHCPv6 服务器获取其它配置参数。

可使用 show ipv6 interface 命令查看当前配置以确定分配方法。如图 2 所示，输出的最后一行表明客户端如何获取地址和其他参数。

图 1 列出了用来在路由器上配置有状态 DHCPv6 的命令。该示例以全局配置模式开始，命令如下所示。第一个命令是 ipv6 unicast-routing。下一命令是 dhcp pool ipv6-stateful。下一命令是，address prefix 2 0 0 1:d b 8:c a f e:1::/64 lifetime infinite。下一命令是 d.n.s.-server 2 0 0 1:d b 8:c a f e:a a a a :: 5。下一命令是 domain-name example.com。已配置 DHCP v6 池，下一组命令将池绑定到接口。示例中的 CLI 首先使用命令 exit 转移到本地接口。Exit 命令之后是 interface g 0/1 命令。下一命令是 i.p.v.6. address 2 0 0 1:d b 8:c a f e:1:: 1/64。下一命令是 dhcp pool ipv6-stateful。最后一个命令 i.p.v.6. n.d. managed-config-flag。突出显示了 address prefix 命令和 ipv6 nd managed 命令，以表示这些命令特定于有状态配置。该图还列出了用来配置无状态 DHCPv6 服务的命令。该示例以全局配置模式开始，命令如下所示。第一个命令是 ipv6 unicast-routing。下一个命令是 dhcp pool ipv6-stateless。下一命令是 d.n.s.-server 2 0 0 1:d b 8:c a f e:a a a a :: 5。下一命令是 domain-name example.com。已配置 DHCP v6 池，下一组命令将池绑定到接口。示例中的 CLI 首先使用命令 exit 转移到本地接口。Exit 命令之后是 interface g 0/1 命令。下一命令是 i.p.v.6. address 2 0 0 1:d b 8:c a f e:1:: 1/64。下一个命令是 dhcp pool ipv6-stateless。最后一个命令是 i.p.v.6. n.d. other-config-flag。在图中突出显示了命令 ipv6 nd other-config-flag，以表示它特定于无状态 DHCPv6 配置。图 2 显示了三种不同配置在特定路由器接口上运行 show ipv6 interface 命令的输出。第一个是 SLAAC 的输出，该图突出显示了指出“hosts use stateless autoconfig for address”（主机使用无状态地址自动配置）的行。接着是无状态 DHCPv6 的输出，该图突出显示了指出“hosts use DHCP to obtain other configuration”（主机使用 DHCP 获取其他配置）的行。第三个是有状态 DHCPv6 的输出，该图突出显示了指出“Hosts use DHCP to obtain routable addresses”（主机使用 DHCP 获取可路由地址）的行。
