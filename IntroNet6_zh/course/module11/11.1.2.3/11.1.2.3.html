基础设施

为了支持现有的和提议的实时应用程序，基础架构必须能适应每种流量类型的特征。网络设计人员必须确定现有交换机和布线是否支持将添加到网络中的流量。

VoIP

VoIP 设备将模拟转换为数字 IP 数据包。此设备可以是附加在传统模拟电话和以太网交换机之间的模拟电话适配器 (ATA)。当信号转换成 IP 数据包后，路由器在相应位置之间发送这些数据包。VoIP 比集成 IP 电话解决方案要低廉得多，但通信质量不能达到相同的标准。例如，使用 Skype 和 Cisco WebEx 的非企业版可以实现小型企业的 IP 语音和 IP 视频解决方案。

IP 电话

在 IP 电话技术中，由 IP 电话机本身执行语音到 IP 的转换，使用集成 IP 电话解决方案的网络中不需要具有语音功能的路由器。IP 电话使用专用服务器实现呼叫控制和信令。现在有许多供应商为小型网络提供专用 IP 电话解决方案。

实时应用程序

为了有效传输流媒体，网络必须能够支持对延时有严格要求的一些应用程序。实时传输协议 (RTP) 和实时传输控制协议 (RTCP) 支持这一要求。RTP 和 RTCP 允许并入服务质量 (QoS) 机制，从而实现网络资源的控制和可扩展性。这些 QoS 机制提供了一些重要的工具，利用这些工具可最大程度地减少实时流应用程序的延时问题。
