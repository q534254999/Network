同一网络中的目的地

将两个主要地址分配给以太网 LAN 上的设备：

  • 物理地址（MAC 地址）– 用于同一网络中的以太网网卡之间的通信。

  • 逻辑地址（IP 地址）– 用于将数据包从原始源发送到最终目的地。

IP 地址用于标识原始源的地址和最终目的地址。目的 IP 地址可在同一 IP 网络中作为源地址，也可用于远程网络。

注意：当给定一个域名（例如 www.cisco.com）时，大多数应用程序会使用 DNS（域名系统）来确定 IP 地址。DNS 将在后面的章节中讨论。

第 2 层或物理地址，和以太网 MAC 地址一样，都有不同的目的。这些地址用于通过封装的 IP 数据包将数据链路帧从同一网络中一个网卡发送到另一个网卡。如果目的 IP 地址处于同一网络中，则目的 MAC 地址是目的设备的 MAC 地址。

图中展示了 PC-A 的以太网 MAC 地址和 IP 地址正在向同一网络中的文件服务器发送 IP 数据包。

第 2 层以太网帧包括：

  • 目的 MAC 地址 – 文件服务器的以太网网卡的 MAC 地址。

  • 源 MAC 地址 – PC-A 的以太网网卡的 MAC 地址。

第 3 层 IP 数据包包括：

  • 源 IP – 原始源，即 PC-A 的 IP 地址。

  • 目的 IP – 最终目的地，即服务器的 IP 地址。

Figure shows a PC connected to a server. The PC is sending data to the server. The PC puts the Layer 3 source and destination IP addresses in the packet header and then puts the Layer 2 source and destination MAC addresses in the frame header.
