使用子网 ID 划分子网

IPv6 全局单播地址的 16 位子网 ID 部分可以由组织用来创建内部子网。

子网 ID 提供足够多的子网和主机支持，比一个子网中需要的还多。例如，16 位部分可以：

  • 创建最多 65,536 /64 个子网。这还不包括从地址的接口 ID 借用位的情况。

  • 每个子网支持多达 18 百亿亿的主机 IPv6 地址（即 18,000,000,000,000,000,000）。

注意：也可以将子网划分为 64 位接口 ID（或主机部分），但是很少这么要求。

IPv6 子网划分也比 IPv4 容易实施，因为不需要转换为二进制。要确定下一个可用子网，只需要将十六进制数相加。

例如，假设一个组织分配了 2001:0DB8:ACAD::/48 全局路由前缀，并采用 16 位子网 ID。这将允许组织创建 /64 子网，如图所示。注意，所有子网的全局路由前缀是相同的。只为每个子网递增子网 ID 的十六位字节数（以十六进制形式）。
